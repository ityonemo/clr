# Null Safety Analysis

## Refinement Types with null_safety

**MUST EXIST (must be non-null)**:
- `optional` - tracks null/non-null state

**MUST BE NULL (has analyte, but null_safety invalid)**:
- `scalar`, `pointer`, `errorunion`, `struct`, `union`, `recursive`, `fnptr`, `allocator`, `region`

**NO ANALYTE (no analyte struct exists)**:
- `void`, `noreturn`, `unimplemented`

NOTE: init() functions may leave these invariants temporarily unsatisfied, as long as the tag handler sets the invariants correctly before the tag processing is complete.

## Purpose

Detects unsafe `.?` (optional unwrap) operations. Tracks null-checking state for optionals to enable type narrowing.

## What It Checks

The `.?` operator is lowered by codegen (`src/`) to:
1. `is_non_null` - runtime safety check
2. `optional_payload` - extract the payload

The `optional_payload` handler checks:
- If there's a preceding `is_non_null` assertion on the same source
- If the optional has been proven `.non_null` via `cond_br`
- If the optional is known `.null` (error: unwrapping known null)

## States

- **unknown** - checked but not yet resolved by cond_br
- **non_null** - known to be non-null (after successful null check in if/while)
- **null** - known to be null

## Limitations

**NOT supported**: Manual null-check patterns like:
```zig
if (opt != null) {
    // use opt.?
} else {
    @panic("null");
}
```

Only the structural `.?` pattern generated by Zig's safety checks is guaranteed to be recognized.
